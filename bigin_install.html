SDK 추적코드 설치하기

시작하기

bigin의 WEB SDK는 웹사이트에 삽입되는 스크립트 형태로, 웹사이트에 접속한 사용자들의 행동 이벤트를 추적하고, 웹사이트로 유입된 사용자의 구매행동 추적 및 다차원 분석을 지원합니다.

웹사이트에서 기본 데이터를 수집하는 방법은 다음과 같습니다.

  1. 새로운 bigin 계정을 만들거나 계정에 로그인합니다.
    • 계정을 만들려면 무료로 시작하기를 클릭합니다.
    • 계정에 로그인하려면 로그인을 클릭합니다.
  2. 프로젝트를 생성하여 추적코드를 발급받습니다.
  3. 안내에 따라 웹사이트 또는 모바일 앱에 추적코드를 추가하면 프로젝트의 기본 추적 데이터가 수집됩니다.

기본 추적 스크립트

  1. 추적 스크립트는 프로젝트마다 고유의 추적 ID로 작동합니다. 프로젝트 생성을 완료하면 bigin.js 를 사용할 수 있는 추적코드가 생성됩니다.
  2. 아래의 설명에 따라 추적 ID를 찾습니다. 추적 코드 스니펫에 이 추적 ID를 사용합니다.
  3. bigin.js 추적 코드 스니펫을 복사하여 추적할 각 웹페이지에 삽입합니다. 각 페이지의 시작 태그인 <head > 바로 다음에 추적 코드 스니펫을 추가합니다.

웹사이트의 SDK의 경우 구글 태그매니저(GTM)의 설치를 지원합니다. 구글 태그매니저의 설치방법은 여기를 참고하세요.

추적 ID 찾기

추적 ID를 찾으려면 다음 단계를 따르세요.

  1. insight.bigin.io에 로그인합니다.
  2. 설정 > 해당 프로젝트를 클릭합니다.
  3. 추적 ID가 페이지 상단에 표시됩니다.

사이트의 각 페이지에서 <head> 태그 바로 뒤에 다음 스니펫을 삽입합니다. 다음과 같이 PROJECTID를 실제 프로젝트의 추적ID로 변경합니다.

추적 코드 스니펫

            
  (function() {
    var biginScript = document.createElement('script');
    biginScript.type = 'text/javascript';
    biginScript.async = true;
    biginScript.src = 'https://sdk.bigin.io/v1/bigin.js'; // bigin SDK 버전
    biginScript.onload = function() {
      bigin.config({
        projectID: "프로젝트 ID", // 프로젝트의 고유 추적 ID 입니다. 'G924TSD55D-2'
        domain: "https://i.bigin.io", // 프로젝트의 도메인 url 입니다.
        track: ['sessions', 'views', 'clicks', 'scrolls'], // 자동 추적(AutoTrack)이 가능한 이벤트와 제스처
        enableTrackLocation:false, // 모바일 접속 시 위치정보 수집 허용
        currencyCode: "KRW" // 표시 통화
      });
    };
    var s = document.getElementsByTagName('script')[0];
    s.parentNode.inserBefore(biginScript, s);
  })();

데이터 수집의 허용

bigin SDK는 선택적으로 기본적인 사용자 행동을 추적할 수 있습니다. 아래에서 선택적으로 추적되는 추적항목을 확인하세요.

            
  track : [
    'session', // 사용자 방문의 단위를 수집하기 위한 단위입니다. 가능한 이 데이터는 추적을 권장합니다.
    'views', // 사용자의 페이지 조회를 수집할 수 있습니다. SPA(SinglePageApplication)은 별도의 설정이 필요합니다.
    'clicks', // 마우스로 클릭하는 모든 클릭이벤트를 수집합니다. 특정 이벤트의 추적은 CustomEvent 의 추적을 이용하세요.
    'scrolls' // 웹사이트 내의 스크롤 이벤트를 수집합니다.
  ]
          
            
  (function() {
    bigin.track('sessions');
    bigin.track('clicks');
    bigin.track('scrolls');
    bigin.track('links');
    bigin.track('forms');
    bigin.track('errors');
  })();

특정 페이지 뷰 추적 사용 중지

이 스니펫의 기본 동작은 bigin SDK 로 페이지 뷰 조회를 보내는 것입니다. 이것은 대부분의 경우 권장하는 부분입니다. 사이트의 각 페이지에 스니펫을 추가하면 페이지 뷰가 자동으로 추적됩니다. 그러나 스니펫이 대시보드로 페이지 뷰 조회를 보내지 않게하려면 send_page_view매개 변수를 false다음 과 같이 설정하십시오 .

              ignore : ['/pagePath', '/pagePath/page'] // 추적 사용을 중지할 특정 페이지의 path를 다음과 같이 추가하세요.

SPA(SinglePageApplication)의 페이지 뷰 추적

SPA(SinglePageApplication) 단일 페이지 어플리케이션으로 개발된 웹사이트의 경우, 페이지 뷰 이벤트의 별도 추적이 필요합니다.

아래의 스니펫을 추적할 페이지의 <head> 태그 안에 추가하세요.

            
  (function() {
    bigin.track('views', {
      page : "/productDetail/12345", // (옵션)pagePath 또는 window.location.pathname 로 추적합니다.
      ignore : ["/main"] // (옵션)추적을 허용하지 않는 페이지의 정규식
    });
  })();

표시 통화의 설정

ISO 4217 기반 화폐단위를 나타내는 코드로 수집 데이터 중 발생하는 수익의 표시 통화를 나타냅니다.

표시통화를 설정하는 방법은 아래와 같으며, 우선 순위의 적용도 다음 순서에 따릅니다.

  1. SDK에서 currencyCode 설정
  2. insight.bigin.io > 설정 > 비즈니스 설정 에서 통화 설정
  3. 기본 설정 = 대한민국 통화(KRW)
            
  (function() {
    bigin.config({
      currencyCode : "KRW"
    });
  })();

고유 사용자 식별

로그인 사용자 식별

bigin은 로그인 또는 회원가입 등 회원의 고유정보를 수집하여, 이커머스 및 웹사이트에서 행동하는 이벤트를 개별 사용자의 행동으로 처리할 수 있도록 사용자를 추적합니다. 로그인을 통한 사용자 중심의 분석은 다양한 비즈니스 접점을 한 명의 사용자 중심으로 이어주기 때문에 전체 사용자의 파악과 경로를 통한 분석을 할 수 있습니다.

WEB SDK 설치 시 아래와 같이 ID 와 EMAIL 을 설정해야 합니다.

            
  (function() {
    bigin.user("profile",{
      'id': 'baf5c122.e9917a0632af', // (필수) bigin에서 발급되는 고유 ID 외에 별도의 ID로 사용자를 식별할 수 있습니다.
      'email': 'support@bigin.io', // (필수) 사용자의 이메일
      'name': '홍길동', // 이름
      'nickname': '길동이', // 닉네임
      'phoneNumber': '01011112222', // 전화번호
      'gender': 'Male', // 성별
      'birth': '1990', // 생년
      'address': '서울시 강남구 선릉로 90길 36 미소빌딩 8층 Big Insight', // 주소
      'property': 'Value', // 회원의 정보에 추가하고 싶은 항목을 함께 추적할 수 있습니다.
    });
  })();

복수 계정 사용자의 추적

한 세션 안에서 로그인하지 않은 상태의 행동과 로그인 후의 행동은 로그인 사용자의 행동으로 통합됩니다.

만약 세션 안에서 한 계정 외에 다른 계정의 로그인이 복수로 발생할 경우 마지막 로그인 사용자의 계정에 이전의 데이터가 모두 속하게 됩니다.

위와 같은 문제를 해결하기 위해서는 로그인을 끊을 수 있는 행동에 아래의 스니펫을 추가하여 기존 로그인 사용자의 세션을 종료하여야 합니다.

            
  (function() {
    bigin.user("logout");
  })();

모바일 사용자의 위치정보 수집

bigin의 기본적인 사용자 위치를 수집하는 방식은 IP에 기반한 주소로 수집합니다. 더 정확한 위치의 수집이 필요한 경우에는 모바일에서 접속한 사용자의 위도와 경도를 수집할 수 있습니다. 아래의 스니펫을 <head> 태그 사이에 추가하세요. 사용자의 위치정보 수집을 묻고, 허용한다면 사용자의 현재 위치의 위도와 경도를 수집하여 저장할 수 있습니다.

            
  (function() {
    bigin.user("location"); // 기본 추적코드에 함께 있는 옵션입니다. 별도 적용 또는 기본추적코드의 enableTrackLocation 을 TRUE 로 허용하세요.
  })();

GTM 설치

시작하기

웹사이트의 SDK의 경우 구글 태그매니저(GTM)의 설치를 지원합니다.

 GTM을 활용한 bigin SDK 설치 방법은 아래와 같습니다.

  1. tagmanager.google.com로 접속 후 태그관리자 계정과 컨테이너를 생성합니다.
  2. 컨테이너를 생성 후, 추적코드를 발급받습니다.
  3. 안내에 따라 웹사이트에 추적코드를 추가하면 프로젝트의 기본 추적 데이터가 수집됩니다.

GTM 기본코드 삽입

가이드에 따라 GTM 추적코드를 프로젝트 내에 삽입합니다.

GTM 변수 생성

data, eventName 이름의 데이터 영역 변수를 생성합니다.

기본 추적 태그 생성

bigin의 모든 GTM 태그는 맞춤 HTML 태그를 사용합니다.

bigin SDK 삽입 태그

bigin SDK 삽입 태그는 페이지뷰 트리거를 사용합니다. 웹사이트 로드 시, 해당 태그가 실행됩니다.

표시 통화의 설정 변경 태그

표시통화를 설정을 변경하는 태그입니다.

표시 통화 설정을 변경하기 위해서 아래의 코드를 프로젝트에 추가합니다.

            
  window.dataLayer.push(
    {'data' : undefined},
    {
      'event' : 'config',
      'data' : {currencyCode : 'KRW'}
    }
  );

SPA(SinglePageApplication)의 페이지뷰 태그

SPA(SinglePageApplication) 단일 페이지 어플리케이션으로 개발된 웹사이트의 경우, 페이지 뷰 이벤트의 별도 추적이 필요합니다.

해당 태그를 실행시키기 위해서 아래의 코드를 프로젝트에 삽입해주어야 합니다.

            
  window.dataLayer.push(
    {'data' : undefined},
    {
      'event' : 'views',
      'data' : {
        'page' : '/productDetail?productId=p12345',
        'ignoreList' : ['/main']
      }
    });
